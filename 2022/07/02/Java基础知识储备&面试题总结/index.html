基础知识

Java语言的特点

  • 简单易学

  • 面向对象

  • 平台无关性（Java虚拟机来实现）

  • 支持多线程

  • 安全可靠

  • 支持网络编程

  • 编译和解释并存

    高级编程语言按照程序的执行方式可以分为两种：

    • 编译型（C 、C++、Go）

      通过编译器，将源代码一次性翻译成平台可执行的机器码。执行速度快，开发效率低。

    • 解释型（Python、JavaScript、PHP）

      通过解释器，将代码一句一句的解释为机器码。开发效率高，执行速度慢。

    [^为了提高编译语言的效率，已经发展出了“即时编译”技术，这种技术缩小了这两种语言的差距，混合了编译语言和解释型语言的优点。实时编译，生成字节码，到执行时将字节码直译、执行。比如说JAVA]:

[^跨平台曾经是JAVA语言最大的优势，目前市面上的虚拟化技术已经非常成熟，我们可以通过各种虚拟化技术来实现java的跨平台，比如说非常简单的docker。现在JAVA最大的优势应该是JAVA的生态！]:

JDK、JRE 和 JVM 的区别和联系

  • JDK：( Java Development Kit ) 是 java 的开发工具。它拥有 JRE 所拥有的一切，还有编译器（javac）和工具（如 javadoc 和 jdb）。它能够创建和编译程序。

  • JRE：( Java Runtime Environment ) java运行时环境。它是运行已编译 Java 程序所需的所有内容的集合，包括 Java 虚拟机（JVM），Java 类库，java 命令和其他的一些基础构件。但是，它不能用于创建新程序。

  • JVM：( Java Virtual Machine) java虚拟机

    Java虚拟机是运行 Java字节码 的虚拟机。JVM 有针对不同系统的特定实现。可以使用相同的字节码来不同的系统出现相同的结果。

    [^JVM 不只有一种]:

什么是字节码

在JAVA中，JVM 可理解的代码叫做字节码（其实就是 .class 文件）

Java 和 C++ 的区别

  • Java不提供指针来直接访问内存，程序内存更加安全
  • Java 的类是单继承的（但是Java的接口可以多继承），C++支持多继承；
  • Java 有自动内存管理垃圾回收机制，不需要程序员手动释放无用内存。
  • C++ 同时支持方法重载和操作符重载，Java 只支持方法重载。（因为操作符重载增加了复杂性，这与Java最初的设计思想不符）
  • 。。。。。。

基础语法

字符型常量和字符串常量的区别

  1. 形式不同。字符型常量是单引号引起的单个字符，字符串常量是双引号引起的多个字符。
  2. 含义不同。字符型常量就是一个ASCII值，字符串常量代表一个地址。
  3. 大小不同。字符型常量占两个字节，字符串常量占若干个字节。

注释有哪几种形式

  1. 单行
  2. 多行
  3. 文档注释

标识符和关键字的区别

  • 标识符我们可以理解为一个名字。
  • 关键字是被赋予特殊含义的标识符。

continue、break 和 return 的区别

  1. continue：跳出当前循环，继续下一次循环。

  2. break：跳出整个循环体，执行循环体下面的语句。

  3. return：跳出当前方法，结束当前方法的运行：

    • return;

      直接使用 return 结束方法执行，用于没有返回值函数的方法。

    • return value;

      return一个特定值，用于有返回值函数的方法。

面向对象的理解

相比与面向过程，面向对象处理问题的角度不一样。面向过程更注重事情每一个步骤及顺序，面向对象更注重事情有哪些参与者、每个参与者需要做什么。面向过程更加的直接高效，而面向对象更易于复用、扩展和维护。

三大特性：

  • 封装

    明确标识出允许外部使用的所有成员函数和数据项
    内部细节对外部调用透明，外部调用无需修改或关系内部实现

    1. JavaBean的属性私有，提供 get set 方法对外访问，因为属性的赋值或者获取逻辑只能由javabean本身觉得，不能由外部胡乱修改。

    2. orm框架

      操作数据库，我们不需要关系链接时如何建立的，sql 是如何执行的，只需要引入 mybatis，调用方法即可。

  • 继承

    继承基类的方法，并作出自己的改变/扩展
    子类共性的方法或属性直接使用父类的，而不需要自己再定义，只需扩展自己个性化的

  • 多态

    基于对象所属类的不同，外部对同一个方法的调用，实际执行的逻辑不同。

    多态的三个条件：继承、方法重写、父类引用指向子类对象。
